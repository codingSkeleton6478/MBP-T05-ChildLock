### Failure_Mode(고장모드) 식별 및 대응 방안 분석

### 🚨 위험 상황 (Hazardous Situation) 정의

| 구분 | 위험 상황 설명 | 환경 및 조건 |
| :---: | :--- | :--- |
| **A** | 주행 중 발생하는 위험 상황 | 속도 > 3km/h, 동적 주행 상태 |
| **B** | 정차/하차 중 발생하는 위험 상황 | 차량 정지, 도어 개방 전후, 외부 객체(자전거 등) 접근 |
| **C** | 전원 상태 변화에 따른 위험 상황 | 전원 OFF, 시스템 리셋, 전원 상실 및 불안정 |

### 🛡️ 차일드 락 소프트웨어 통합 안전 분석 (FMEA/HARA 기반)

| 유스케이스 (UC) | 구분 | 위험 시나리오 | Failure Mode (고장 유형) | 통합 고장 상황 상세 | 위험도 | 통합 대응 방안 (Safety Design & SW Logic) |
| :--- | :---: | :--- | :--- | :--- | :---: | :--- |
| **UC-1**<br>운전자 차일드락 전자식 ON/OFF 제어 | **A** | 주행 중 차일드락 해제 요청 | Unintended Unlock<br>(의도되지 않은 기능) | 주행 중(>3km/h) 실수로 버튼이 눌려 차일드 락이 해제됨 | **상** | **속도 센서 값 기반 인터락:** 속도 > 3km/h 조건에서 해제 요청 차단.<br>**입력 검증:** 의도 확인 입력(2초 롱프레스 또는 2회 입력) 로직 및 스위치 입력 디바운싱 적용 |
| | **C** | 전원 리셋 후 상태 손실 | State Loss<br>(상태 손실) | 전원 리셋으로 인해 기존 차일드 락 설정 상태 정보를 상실함 | **중** | **비휘발성 메모리 동기화:** 직전 상태를 NVM에 저장하고, IGN ON 시 복구하여 상태 재동기화 수행 |
| **UC-2**<br>차량 출발 시 자동 차일드락 활성화 | **A** | 주행 중 자동 활성화 미동작 | Auto Lock Missing<br>(기능 손실) | 속도 임계 초과 및 주행 중임에도 자동 차일드 락 ON 기능이 동작하지 않음 | **상** | **속도 신호 이중화/Fail-safe:** ABS/Wheel speed 이중화 검증, 신호 이상 시 CL ON 유지. 임계 초과 시 강제 ON 수행 및 상태 검증. DTC 및 경고 표시 |
| | **C** | 전원 리셋 후 자동 기능 비활성 | Mode Reset Error<br>(기능 비활성) | 전원 리셋 후 자동 잠금 설정값이 초기화되어 기능이 비활성화됨 | **중** | **동적 상태 재평가:** NVM에서 설정값 복구 후 현재 속도를 기반으로 상태 재평가 로직 수행 |
| **UC-3**<br>사고 발생 시 차일드락 자동 해제 | **A** | 충돌 신호 오탐 | False Crash Detection<br>(오작동) | 에어백 미전개 등 실제 충돌이 아닌 상황에서 신호 오탐으로 해제됨 | **중** | **충돌 검증 고도화:** 신뢰도 높은 조건(에어백 전개 등) 기반 충돌 검증 로직 적용 |
| | **C** | 사고 중 전원 불안정으로 해제 실패 | Emergency Unlock Failure<br>(기능 손실) | 제어기 전원 상실/불안정으로 충돌 후 차일드 락 해제가 불가하여 승객 탈출 불가 | **상** | **다중 해제 경로 확보:** 전원 상실 대비 백업 전원/기계식 해제 경로 확보. 충돌 감지 시 Unlock 명령 최우선 처리, 최소 1회 강제 해제 및 명령 재전송 로직 적용 (안전 MCU 활용) |
| **UC-4**<br>뒷좌석 내부 문 열기 시도 차단 | **A** | 주행 중 내부 문 열기 시도 | Door Open Attempt<br>(부분 기능 상실) | CL ON 상태에서 일부 도어만 차단되고 개방 시도가 허용되거나 차단에 실패함 | **상** | **폐루프(Closed-loop) 제어:** 내부 개방 요청 차단 로직 적용 및 도어별 상태 피드백 확인. 미일치 시 재명령 수행, 이벤트 로그 기록 및 운전자 경고/진단 코드 기록 |
| | **C** | 전원 복구 후 상태 불일치 | Inconsistent State<br>(상태 불일치) | 전원 복구 후 제어기 시스템과 실제 도어 래치 상태가 불일치함 | **중** | **상태 동기화:** 전원 복구 시 Door ECU 상태 조회 후 즉시 동기화 수행 |
| **UC-5**<br>후방 위험 접근 보호 | **B** | 정차 중 후방 접근 위험 | Dooring Risk<br>(기능 저하) | 자전거 등 후방 위험 접근 감지가 늦거나 센서 신뢰도가 낮아 경고 및 보호가 적시 수행되지 않음 | **상** | **센서 융합 및 보수적 제어:** 레이더+카메라 센서 융합, 거리+접근속도 위험 판단 알고리즘 적용. 신뢰도 저하 시 보수적 전략(CL 유지), 위험 시 CL 자동 ON, 센서 성능 자가진단 수행 |
| | **C** | 전원 리셋 후 보호 기능 비활성 | Protection Disabled<br>(보호 해제) | 전원 리셋 과정에서 센서 초기화 지연으로 기능이 비활성화됨 | **중** | **센서 재점검 로직:** 전원 복구 시 센서 상태 우선 점검 및 기능 재활성 로직 수행 |
| **UC-6**<br>뒷좌석 점유 기반 출발 알림 | **A** | 점유 있음에도 출발 시 알림 누락 | Missing Notification<br>(지연된 기능) | 점유 상태임에도 출발 시 알림이 누락되거나 일정 시간 경과 후 지연 표시됨 | **중** | **실시간 알림 및 다중 채널:** 출발 이벤트 즉시 1초 이내 알림 요구조건 설정. 점유 상태+CL 상태 검증 후 1회 알림. HMI Watchdog 및 대체 경고 채널(경고음) 병행 |
| | **C** | 전원 리셋 후 점유 상태 손실 | Occupancy State Loss<br>(상태 손실) | 리셋으로 인해 뒷좌석 승객 점유 센서 정보가 상실됨 | **하** | **센서 재확인:** IGN ON 시 뒷좌석 점유 센서 재확인 후 상태 즉각 반영 |
| **UC-7**<br>시동 OFF 시 차일드락 상태 알림 | **B** | 정차 중 CL ON 상태 인지 부족 및 과도 경고 | State Unaware<br>(초과 기능) | 시동 OFF 시 CL 상태 인지가 부족하거나, 하차 시 알림이 과도하게 반복되어 타 위험 경고(BSD 등)와 충돌함 | **하** | **우선순위 제어 알고리즘:** 시동 OFF 시 CL ON 상태 요약 메시지 표시. HMI 메시지 우선순위 제어(타 위험 경고 우선), 알림 1회/시간 제한 로직 적용 |
| | **C** | 전원 종료 직전 알림 미표시 | Notify Failure<br>(기능 누락) | 시스템 전원 종료 시퀀스로 인해 알림 메시지 출력이 누락됨 | **하** | **종료 시퀀 개입:** IGN OFF 이벤트 감지 즉시 상태 조회 프로세스를 우선 할당하여 1회 표시 수행 후 종료 |
